    13. 문서의 구조와 슈퍼스타들

  1. 1. 모든 웹페이지들의 공통적인 구조: html, head, body

    1. html: 최상위 태그. 모든 코드와 태그들은 다 이 html 태그 안에 묶인다.
    2. body: 본문을 나타내는 태그. 이 안에 적힌 내용은 페이지에 표시된다.
    3. head: 본문을 설명하는 태그. 이 안에 적힌 내용은 페이지에 직접적으로 표시되진 않아도, 본문의 속성 등을 나타낸다.
    =>즉 모든 태그들은 html에 속하며, head와 body태그 중 하나에도 무조건 속한다.

    2. head 태그 관련된 부가정보

    meta 라는 태그가 있다. 이 태그는 주로 속성 등 본문을 설명하므로 head 태그 안에 쓰인다.
    ex) meta 태그에는 charset(=character set)이라는 속성이 많이 쓰인다. 이는 '문자 코딩 방식'을 의미한다.
    charset 방식엔 여러 가지가 있는데, 이중 한글도 코딩 가능한 방식이 'utf-8' 이다. meta charset을 utf-8로 설정하지 않으면 한글을 코딩할 수 없다.

    17. 서버와 클라이언트

    서버와 클라이언트의 개념

    서버 : Web server 가 있는 컴퓨터 클라이언트 : Web browser 가 있는 컴퓨터 대부분 기본 컴퓨터에는 웹 브라우저가 깔려 있으므로, 대부분의 컴퓨터는 클라이언트이다! => 그렇다면 웹 서버는 무엇인가?

    서버와 클라이언트가 상호작용하는 원리

    1. 클라이언트가 주소창에 어떤 홈페이지 주소를 입력하고 클릭을 누른다.
    = 클라이언트는 해당 주소에 해당하는 웹페이지를 보여줄 것을 서버에 요청한 것이다.

    2. 웹 서버는 클라이언트가 요청한 웹페이지를 포함한, 여러 웹페이지들의 코드를 컴퓨터에 가지고 있다.
    -> 서버 컴퓨터는 서버 안에 있는, 클라이언트가 요청한 주소의 웹페이지에 해당하는 코드를 불러온다.

    3. 해당 코드에 대한 정보가 담긴 신호를 다시 클라이언트 컴퓨터로 전송한다.

    4. 클라이언트 컴퓨터는 그 신호를 받아서 자신의 웹브라우저에 보낸다. 그러면 웹브라우저는 해당 코드를 받아서 웹페이지의 모습으로 구현한다.
